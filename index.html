first_query.py 
from cdf_client import CdfClient, ApiKeyAuth

# Connect to your CDF cluster
client = CdfClient("http://localhost:8080",
                   auth=ApiKeyAuth("your-api-key"))

# Insert a document with embedding
client.insert("papers", {
    "title": "Attention Is All You Need",
    "embedding": {"model_id": "all-MiniLM-L6-v2",
                    "values": [0.1, 0.2, ...]}
})

# Semantic search + graph traversal
results = client.search("papers", vector=[0.1, 0.2, ...], top_k=5)
paths = client.traverse("paper-001", edge_types=["cites"], depth=2)

Our Story

Why We Built CDF

🎯 Mission

To build the foundational data infrastructure for the AI-native era — where semantic meaning, relationships, time, and uncertainty are first-class citizens, not afterthoughts.

🔭 Vision

A world where developers spend zero time wiring together databases and 100% of their time building intelligent applications that understand context, reason over relationships, and handle uncertainty gracefully.

The Problem We're Solving

❌
Vector DBs

Pinecone, Weaviate, Milvus — no relational joins or graph traversal

❌
Graph DBs

Neo4j, TigerGraph — no native vector similarity search

❌
Relational DBs

PostgreSQL + pgvector — embeddings are bolted on, slow at scale

⬇
✅
Cognitive Data Fabric

One engine: vectors + graphs + temporal + uncertainty, natively integrated

What Makes CDF Different

🧬

Poly-Modal Values

One row holds scalars, embeddings, tensors, distributions, blobs, and graph edges together.

⏰

Bitemporal by Default

Every record tracks both valid-time (reality) and transaction-time (system) automatically.

🎲

Native Uncertainty

Confidence scores and probability distributions are built into the type system, not metadata.

🔗

Semantic Graph

Vectors and relationships coexist — query by similarity AND traverse connections in one query.
